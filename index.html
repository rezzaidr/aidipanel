// architecture

One less service.
Zero compromise.

CloudPanel routes traffic through Varnish before Nginx. AidiPanel removes that layer entirely — FastCGI Cache lives inside Nginx itself.

CloudPanel
Request flow
Browser
↓
Nginx (port 80)
↓
Varnish (extra service)
↓
PHP-FPM
↓
MySQL + Redis
AidiPanel
Request flow
Browser
↓
Nginx + FastCGI Cache ← hit? serve instantly
↓ miss only
PHP-FPM
↓
MariaDB + Redis
✓ No Varnish daemon to manage
✓ Cache config is standard Nginx directives
✓ Lower RAM usage (~50MB saved)
✓ Simpler troubleshooting
✓ Redis still handles object cache
// features

Everything you need.
Nothing you don't.

⚡
FastCGI Cache
Native Nginx caching with per-site toggle, URL purge, and hit-rate monitoring. No VCL, no separate service.
🔴
Redis Object Cache
Redis bound to localhost for WordPress object cache, session storage, and application-level caching.
🐘
Multi-PHP
PHP 8.1, 8.2, and 8.3 installed simultaneously. Switch per site from the panel or CLI in seconds.
🗄️
5 DB Engines
Choose from MariaDB 10.11 / 11.4 / 11.8 or MySQL 8.0 / 8.4 at install time. MariaDB default for WordPress.
🔒
Auto SSL
Let's Encrypt certificates installed with one command. Auto-renewal via cron. Self-signed for immediate setup.
🛡️
Security Built-in
UFW firewall, Fail2ban, CSRF protection, session hardening, and brute-force throttle out of the box.
💻
Web Panel UI
Clean PHP-based panel. No framework bloat — just Router + SQLite + Alpine.js. Runs on port 8443 HTTPS.
⌨️
CLI Tool
25 commands to manage sites, cache, databases, PHP, SSL, services, and users from SSH.
💾
Auto Swap
Installer auto-creates 2GB swap if none exists. Adaptive sizing based on available RAM. Kernel tuning included.

// installation

Up in minutes.
Really.

Fresh VPS → working panel in under 10 minutes on most servers.

STEP 01
Run the installer
bash <(curl -fsSL \
  https://raw.githubusercontent.com/\
  rezzaidr/aidipanel/main/install.sh)
Supports Debian 11/12 and Ubuntu 22.04/24.04 on x86_64 & ARM64.
STEP 02
Deploy panel app
scp -r aidipanel-app/ \
  root@server:~/
cd ~/aidipanel-app
bash deploy-panel.sh
The web panel is separate so you can update it independently of the server stack.
STEP 03
Add your first site
aidipanel site:add \
  --domain example.com \
  --type wordpress \
  --php 8.3
Then install SSL: aidipanel ssl:install --domain example.com
// cli tool

Manage everything
from SSH.

25 commands covering sites, cache, databases, PHP, SSL, services, and users.

root@vps-prod-01 ~
